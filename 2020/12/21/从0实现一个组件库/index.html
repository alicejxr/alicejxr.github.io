Icons 组件库

对于 Icons 组件我们常见的处理方式有以下几种：

  1. React SVG Component
  2. iconfont
  3. SVGUseElement
  4. 直接使用SVG元素
  5. 使用 base64

对于目前项目，希望 Icons 组件可以满足以下要求：

  1. 按需加载
  2. 可复用
  3. 可自定义 Svg 元素样式
  4. 不同分辨率屏幕下都有较好的清晰度
  5. 可修改 svg 内部元素

根据以上5点需求，克总结以下表格：

按需加载 不同分辨率屏幕下都有较好的清晰度 可复用 样式可自定义 可修改内部SVG元素
React SVG Component 支持 支持 支持 支持 支持
iconfont 不支持 会有失真 支持 部分支持 不支持
SVGUseElement 支持 支持 不支持 支持 支持
直接使用 SVG 元素 支持 支持 不支持 支持 支持
使用 base64 格式图片 支持 支持 不支持 部分支持 不支持

综上选择了 React SVG Component 来实现 Icons 组件，主要使用了 svgr 来生成 React SVG 组件，使我们可以像使用普通 React 组件一样使用 Icon，我们只需要导入经过压缩并且格式正确的 svg，并运行 npx @svgr/cli -d ./src/components/Icons ./assets/svgs 将 ./assets/svgs 下的 SVG 生成 React 组件到 ./src/components/Icons ，并生成 index 导出文件，方便导出。

jest+enzyme自动化测试

jest 结合 enzyme 进行组件的自动化测试，配置参考：jest 、eazyme

为组件写单元测试

测试文件统一写在 __test__ 文件下，以 Button 组件为例

执行单元测试

配置及编写完成单元测试，运行 npm run jest 即可运行单元测试

代码提交及格式化规范

提交代码时，我们希望可以实现：

  1. 组件单元测试执行及代码格式化
  2. message 格式化

基于以上两点我们使用 husky 来在 git pre-commit hooks 来进行组件单元测试执行及代码格式化，在 commit-msg hooks 进行 message 格式化，使用 @commitlint/config-angular 规范

组件打包

TypeScript 生成类型文件

tsconfig.build.json 配置文件

package.json 添加 scripts "build:types": "tsc -p tsconfig.build.json"

组件打包

组件打包有两套方案：一套是 es module 模块化方案，一套是 common module 模块化方案。

组件的打包选择使用 gulp 分别处理样式文件和 TypeScript 文件，gulp 配置文件为 gulpfile.js

TypeScript 文件打包配置

由于需要生成两套不同打包方式的代码，我们需要用到 babel 来帮助我们，将代码编译为符合不同模块化规范的代码。

样式文件打包配置

针对样式文件，由于已有项目都是使用 less，所以并没有将 less 转换为 css

执行 gulp task

将我们写好的 gulp task 组合起来，并运行 gulp

添加 build scripts

在 package.json 的 scripts 中添加

  1. "npm run clean": "rimraf lib es" 删除上次生成的打包好的组件
  2. npm run clean && npm run build:types && gulp 打包最新的组件

npm 发布

release 脚本

并在 package.json 中添加 scripts "publish": "ts-node release.ts"

每次发布组件库时，只需在 master 分支执行 npm run publish 即可发布

后续优化
