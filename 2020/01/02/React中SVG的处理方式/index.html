React中SVG的处理方式

发表于 2020-01-02 更新于 2021-01-06 分类于 svg 阅读次数：

svg 与 jpg、png 等一样是一种图片格式，全称是：Scalable Vector Graphics（可缩放矢量图形）。

为什么要使用 SVG 图片格式

与经常使用的 jpg、png 等图片格式相比， svg 的优势大致有以下几点：

  1. 可压缩性更新强，
  2. 可伸缩性强，不会像其他格式图片放大图片质量就会下降（表现为放大后图片模糊）
  3. SVG 是基于 XML 的标记语言，可以像 HTML 一样被 javaScript 操作，可以创造出更丰富的交互
  4. 可以使用任何文本编辑器和绘图软件来创建和编辑
  5. SVG 中的文字可以直接被搜索，相比普通图片有更好的 SEO 效果。

在 React 中常见的 SVG 使用方式

直接使用 SVG 原始文件

对于每个 SVG 都需要手动拷贝，并且需要手动压缩。当需要导入的 SVG 很多时，效率就会很低。

使用 SVG Sprite

SVG Sprite 是基于 SVG 中的 svg、use、symbol 三种元素来实现的。可以简单的归纳为 symbol + use => SVG Sprite

在一个 svg 文件中

通过在 symbol 元素内来定义需要重复使用的图形元素，通过 use 元素来使用定义好的图形元素（只需要确保 symbol 中的 id 与 use 中的 xlink:herf 保持一致即可）。需要注意一点的是：symbol 中的元素本身是不会呈现在页面上，必须使用 use 元素来实例化（引用了 symbol 元素的 use 元素）后才可呈现。

SVG Sprite 就是通过将 SVG 定义在 symbol 元素中，并将所有的元素存在一个文档中，我们在其他文件中使用 use 元素就可以调用已经定义好的 SVG 元素。

具体实现是：在页面初始化时，将 SVG Symbols 作为 body 的的第一个元素插入 HTML 中以供使用。

这样就减少了图片请求的数量，并且每个图片都可以更改大小、颜色。使用起来非常方便。但同时，也加载了当前页面使用不到的图片，不能按需加载。

按需加载的 SVG 组件

其实就是将手动复制粘贴使用 SVG 图片的过程自动化，通过执行命令，将 SVG 转化为 React 组件，并且统一 export 出来。可以像组件一样使用 import 来引入，并且可以按需加载。svgr 已经帮我们实现了这些。在下一篇文章将分析它的源码，了解整个处理过程。
