React hooks（useState、useEffeect） 实现原理

发表于 2020-12-14 更新于 2021-01-06 分类于 react

前言

目前所有使用 React 进行开发的项目都用上了 React Hooks，虽然知道 useState 可以用来替代 class component 中的 state， useEffect 可以替代生命周期函数，但是它们在原理和实现上还是有很大不同的，接下来将通过模拟实现这两个 hooks，来具体理解它们之间的区别

在模拟实现 useEffect 之前我们需要看一下它的以下特点：

  1. 有两个参数，第一个参数是一个 callback 函数；第二个参数是一个数组（依赖数组）
  2. 当依赖数组不为空时，每当数组中的变量值发生改变时，会执行callback 函数
  3. 当依赖数组是一个空数组时，callback 函数只执行一次
  4. 当依赖数组不传时，每次渲染都会执行 callback 函数

进阶

上面我们简单的实现了 useState 和 useEffect，但是当我们在一个组件内使用多个 useState 和 useEffect 时，上面的写法就会出现问题，我们接下来需要处理多个 useState 和 useEffect 的情况

那如何处理多个 state 的情况呢？我们可以通过 数组 来实现。关键有一下几点：

  1. 首次渲染时，按照 useState、useEffect 的顺序，将 state 和 deps（依赖数组）依次存入 memoizedState 数组中
  2. 更新时，依次从 memoizedState 中将值取出

在线调试：https://codesandbox.io/s/react-usestate-useeffect-simulate-cpulp?file=/src/index.js

以上我们实现了一个可多次重复使用的 useState 和 useEffect，为了便于理解我们使用数组来实现，但是 React Hooks 的源码中的实现，却有所区别。

React Hooks 的真正实现

  1. 使用 单链表 替代 数组，通过链表来链接所有 hooks
  2. 在 React 中，整个应用对应的是一个 Fiber 树，树中的每一个节点对应一个组件，hooks 的数据就作为组件信息的一部分存储在对应节点上，伴随组件的初始化到销毁。
